ModCallbacks
Callbacks to call functions on given events

Fields:

  • MC_NPC_UPDATE Triggered upon an NPC update (not every frame)
  • MC_POST_UPDATE Triggered upon player update (not every frame)
  • MC_POST_RENDER Triggered after a render finishes. Graphical rendering should be done here
  • MC_USE_ITEM Triggered when the player uses an Active Item (e.g. Tammyâ€™s Head)
  • MC_POST_PEFFECT_UPDATE UNK
  • MC_USE_CARD Triggered when the player uses a card
  • MC_FAMILIAR_UPDATE Triggered upon a familiars update (not every frame)
  • MC_FAMILIAR_INIT Triggered when a familiar is constructed and initialised (effectively, upon loading a game with a familiar, or getting a new one)
  • MC_EVALUATE_CACHE Triggered when the cache changes. This primarily means upon stat changes
  • MC_POST_PLAYER_INIT Triggered after the player is constructed and initialised (Game Start)
  • MC_USE_PILL Triggered when the player uses a pill
  • MC_ENTITY_TAKE_DMG Triggered when any Entity takes damage
  • MC_POST_CURSE_EVAL Triggered after the current floors curse is decided
  • MC_INPUT_ACTION Triggered upon input
  • MC_LEVEL_GENERATOR (NOT IMPLEMENTED)
  • MC_POST_GAME_STARTED Triggered once the game loads, and fades in
  • MC_POST_GAME_END Triggered after exit to menu, should be used for saving data
  • MC_PRE_GAME_EXIT Triggered after exit button is pressed, but before game exits
  • MC_POST_NEW_LEVEL Triggered on a new ROOM. This is likely to change
  • MC_POST_NEW_ROOM (NOT IMPLEMENTED)
  • MC_GET_CARD Triggered after a player picks up a card
  • MC_GET_SHADER_PARAMS Assumedly triggered whenever a shader is called
  • MC_EXECUTE_CMD Triggered upon executing a debug console command
